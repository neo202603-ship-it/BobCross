# 밥크로스 앱 매력도 개선 제안

작성일: 2026-05-30

## 배경

밥크로스(Bab-Cross)는 서버 없이 가까운 Android 기기끼리 연결해 점심 메뉴를 정하는 앱이다. 현재 앱은 Nearby Connections 기반 밥판 열기, 초대 수락, 메뉴 투표, 결과 공유, 밥닉네임/아바타, 템플릿, 결과 기록, 영수증/해시 검증, 혼밥 테스트를 갖추고 있다.

더 많은 사람들에게 어필하려면 기능을 무겁게 늘리기보다, 처음 30초 안에 재미와 쓸모가 바로 느껴지게 만드는 방향이 효과적이다.

## 핵심 방향

- 첫 사용자가 앱의 목적과 흐름을 즉시 이해하게 만든다.
- 주변 기기가 없어도 앱의 재미를 체험할 수 있게 한다.
- 결과 화면을 공유하고 싶은 형태로 만든다.
- 밥크로스만의 말투와 마스코트 세계관을 더 살린다.
- 권한 요청과 근거리 연결에 대한 불안을 줄인다.

## 우선순위 높은 개선안

### 1. 첫 실행 온보딩 강화

Nearby 권한, 자동 연결, 밥판 열기 개념은 처음 보는 사용자에게 낯설 수 있다. 첫 실행 시 3단계 온보딩을 제공하면 진입 장벽을 낮출 수 있다.

권장 흐름:

1. 밥친구 찾기
2. 후보 고르기
3. 결과 공개

온보딩 문구는 기능 설명보다 상황 중심으로 구성한다.

예시:

- "메뉴 못 정하는 사람끼리 가까이 모이면 밥판이 열립니다."
- "각자 먹고 싶은 메뉴를 고르고, 시간이 끝나면 결과가 공개됩니다."
- "위치 추적이 아니라 근처 밥친구를 찾기 위한 권한만 사용합니다."

### 2. 혼밥 테스트를 체험 모드로 승격

현재 `혼밥 테스트`는 개발자 진단 성격이 강하다. 이를 홈 화면의 "체험해보기" 기능으로 전면 배치하면 주변 기기 없이도 앱의 핵심 경험을 바로 보여줄 수 있다.

권장 변경:

- 홈에 "혼자 체험하기" 또는 "밥판 체험하기" 버튼 추가
- 가상의 밥친구 2~4명이 참여한 것처럼 투표 흐름 시뮬레이션
- 결과 팝업과 결과 카드까지 실제 흐름처럼 제공

기대 효과:

- Play Store 설치 직후 이탈 감소
- 주변에 같이 테스트할 사람이 없어도 앱의 재미 전달
- 스크린샷과 리뷰 소재 확보

### 3. 밥판 템플릿을 상황형으로 확장

현재 템플릿 기능은 좋은 기반이다. 이를 상황형, 유머형 템플릿으로 확장하면 앱의 개성이 더 살아난다.

추천 템플릿:

- 팀장님 카드 찬스
- 비 오는 날 국물전
- 다이어트 하는 척
- 5분 안에 정해야 함
- 회식 2차 안 가는 메뉴
- 어제 먹은 거 금지
- 맵찔이 보호 구역
- 배달비 아끼는 밥판

템플릿에는 질문, 기본 후보, 제한 시간, 선택 공개 여부를 함께 저장한다.

### 4. 결과 화면의 공유성 강화

결과 화면을 "오늘의 밥결정 카드"처럼 구성하면 사용자가 자연스럽게 캡처해 공유할 수 있다.

포함 요소:

- 우승 메뉴
- 득표율 또는 득표수
- 참여자 아바타
- 밥해시 확인 상태
- 한 줄 멘트
- 밥크로스 마스코트

예시 문구:

- "오늘의 밥결정: 돈까스"
- "3표 중 2표로 밥상 등극"
- "어때요, 메뉴 고르기 참 쉽죠?"

기대 효과:

- 앱 외부 공유 가능성 증가
- 브랜드 인지도 강화
- Play Store 스크린샷 품질 향상

### 5. 권한 설명을 더 친근하게 개선

Bluetooth, Wi-Fi, 위치 권한은 사용자에게 부담스럽게 보일 수 있다. 권한 요청 전에 앱 내부 안내를 먼저 보여주는 것이 좋다.

권장 문구:

- "밥크로스는 서버 없이 가까운 기기끼리만 연결합니다."
- "위치 기록을 저장하거나 추적하지 않습니다."
- "권한은 근처 밥친구를 찾는 데만 사용됩니다."

권한 거부 시에도 단순 오류 대신 체험 모드로 유도한다.

## 추가 매력 기능

### 메뉴 추천 룰렛

후보 자체를 고르지 못하는 사용자를 위해 "아무거나 3개 넣기" 또는 "룰렛으로 후보 추가" 기능을 제공한다.

추천 후보는 상황형으로 나눌 수 있다.

- 점심 빠르게
- 든든하게
- 가볍게
- 비 오는 날
- 야근 전용
- 후식

### 취향 태그

후보에 간단한 태그를 붙이면 결정이 더 현실적으로 느껴진다.

예시 태그:

- 매운맛
- 가성비
- 빨리 나옴
- 든든함
- 면
- 밥
- 배달 가능
- 웨이팅 주의

### 오늘의 밥친구 기록

결과 기록을 단순 목록에서 가벼운 통계로 확장한다.

예시:

- 이번 주 가장 많이 이긴 메뉴
- 가장 자주 같이 먹은 밥친구
- 3연속 국밥
- OO님은 항상 돈까스 편

### 동점 처리 이벤트

동점이 나왔을 때 단순 랜덤 대신 앱의 캐릭터를 살리는 이벤트를 넣는다.

예시:

- 운명의 밥공기 재투표
- 10초 결승전
- 마스코트의 최종 선택

### 지도 검색 연결

결정된 메뉴를 외부 지도 앱 검색으로 연결하면 실사용 가치가 커진다.

초기 구현은 무겁게 만들 필요 없이 다음 정도가 적절하다.

- 결정된 메뉴로 네이버 지도 검색 열기
- 결정된 메뉴로 카카오맵 검색 열기
- Android 기본 지도 검색 인텐트 제공

## 브랜딩 및 문구 개선

밥크로스는 이름과 세계관이 기억에 남는 앱이다. 마스코트와 밥판 용어를 더 적극적으로 쓰는 것이 좋다.

권장 방향:

- 밥공기 마스코트를 홈, 결과, 빈 상태, 오류 상태에 더 자주 노출
- 버튼과 화면 제목을 밥크로스식으로 통일
- Play Store 스크린샷은 기능 설명보다 "메뉴 못 정하는 순간"을 보여주기

문구 예시:

- 밥판 열기
- 밥신호 보내기
- 결승 밥상
- 오늘의 메뉴 확정
- 밥친구 찾는 중
- 밥상이 열렸습니다

## 권장 1차 실행 순서

1. `혼밥 테스트`를 홈의 "체험해보기"로 승격
2. 결과 화면을 공유하고 싶은 카드형으로 개선
3. 템플릿을 상황형/재미형으로 확장
4. 권한 요청 전 안내 화면 추가
5. 결정된 메뉴로 지도 검색 열기 추가

이 조합은 개발 부담이 비교적 작으면서도, 처음 써보는 사용자에게 "귀엽고 쓸모 있다"는 인상을 주기 좋다.
