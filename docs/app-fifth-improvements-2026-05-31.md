# 밥크로스 5차 개선안

작성일: 2026-05-31

## 현재 기준

- 최신 커밋: `5a9aebb Clarify poll invitation and template copy`
- 최신 배포 버전: `0.1.32 / versionCode 33`
- 4차 개선안: `docs/app-fourth-improvements-2026-05-30.md`

최근 4차 개선 흐름에서 다음 항목은 이미 반영되었다.

- 밥판 열기 화면의 `혼자 먼저 정리` / `같이 고르기` 모드
- 공유 카드 미리보기
- 공유 이미지 안전 영역과 긴 텍스트 처리
- 설정 화면의 밥판 기본값
- `개발자 진단`을 `밥친구 찾기 점검`으로 정리
- 밥판 초대 화면의 `이 밥판 규칙` 카드
- 규칙 카드 남은 시간 배지와 상단 타이머 동기화
- 기본 템플릿 질문의 일상 말투 정리

따라서 5차 개선은 같은 기능을 다시 늘리는 대신, 실제 단말기에서 반복 사용했을 때 남는 어색함을 줄이고 출시 전에 제품의 사용 맥락을 더 또렷하게 만드는 데 둔다.

## 기존 개선안에서 피할 중복

1-4차 개선안은 다음 축을 이미 충분히 다뤘다.

- 첫 실행 온보딩과 체험 모드 강화
- 결과 카드 공유성과 이미지 안전 영역
- 상황형 템플릿과 메뉴 룰렛
- 밥판 기본값과 설정 화면 구조
- 연결 문제를 사용자 언어로 바꾸는 작업
- 혼자 먼저 정리하는 밥판장 흐름
- 초대받은 사람에게 보이는 규칙 요약

5차에서는 위 항목을 새로 제안하지 않는다. 다만 이미 구현된 기능이 실제 상황에서 더 자연스럽게 이어지도록 `마지막 20%`를 다듬는다.

## 5차 목표

밥크로스의 다음 목표는 다음 문장으로 정리한다.

```text
처음 켠 사람은 덜 설명하고, 초대받은 사람은 덜 망설이고, 결정 뒤에는 바로 움직이게 한다.
```

현재 앱은 밥판을 만들고 공유하는 구조가 갖춰졌다. 이제 중요한 것은 사용자가 앱 안에서 무엇을 눌렀는지보다, 앱 밖의 점심 대화가 실제로 짧아지는지다.

5차 개선의 판단 기준은 다음과 같다.

- 첫 사용자가 `이 앱을 왜 켜야 하는지`를 화면 전환 없이 이해하는가
- 밥판장이 결과를 만들고 나서 다음 행동까지 자연스럽게 이어지는가
- 초대받은 사람이 권한, 연결, 공개 여부 때문에 멈추지 않는가
- 반복 사용자가 매번 같은 세팅과 후보 입력을 덜 하는가
- Play Store 출시 전에 스크린샷, 권한 설명, 예외 상황이 검증 가능한가

## 우선순위 높은 개선안

### 1. 밥판 열기 화면에 `상대에게 이렇게 보여요` 미니 프리뷰 추가

`혼자 먼저 정리`와 `같이 고르기` 모드가 생기면서 밥판장의 목적은 더 분명해졌다. 하지만 처음 쓰는 사람은 여전히 `밥신호를 보내면 상대 화면에 뭐가 뜨지?`를 상상해야 한다.

밥판 열기 화면 하단 또는 설정 영역 위에 작은 초대 카드 프리뷰를 둔다.

포함 정보:

- 질문 제목
- 후보 개수
- 남은 시간 또는 제한시간
- 선택 공개 여부
- 후보 추가 가능 여부
- 상대가 볼 주요 버튼 문구

예시 문구:

- `친구에게는 이렇게 보여요`
- `김치찌개 vs 돈까스 밥판`
- `후보 5개 · 3분 안에 고르기`
- `내 선택은 결과에서만 보여요`

기대 효과:

- 밥판장이 초대 전 불안감을 덜 느낀다.
- 초대받는 사람의 경험을 미리 알 수 있어 기능 학습이 빨라진다.
- Play Store 스크린샷에서 앱의 핵심 흐름을 한 장에 보여줄 수 있다.

구현 메모:

- 새 화면을 만들기보다 기존 밥판 열기 화면 안에 읽기 전용 카드로 추가한다.
- 현재 입력값이 바뀌면 프리뷰도 즉시 갱신한다.
- 긴 질문은 2줄, 후보 개수는 숫자로 요약한다.
- `혼자 먼저 정리` 모드에서는 `결과 카드로 공유돼요` 프리뷰로 문구를 바꾼다.

### 2. 결과 화면에 `이제 어디로 갈까` 후속 행동 묶음 추가

밥크로스는 메뉴를 정하는 앱이지만, 실제 점심 흐름에서는 메뉴가 정해진 뒤 `그럼 어디 가지?`가 바로 이어진다. 지도 기능을 크게 만들 필요는 없지만, 결정 직후 행동을 묶어주면 제품의 쓸모가 확 좋아진다.

결과 화면의 공유 카드 아래에 `이제 어디로 갈까` 액션 묶음을 추가한다.

권장 액션:

- `지도에서 찾기`
- `대화방에 보내기`
- `이 후보로 다시 열기`
- `지난 결정에 저장됨` 상태 표시

초기 구현은 Android 인텐트 기반으로 충분하다.

- 결정 메뉴로 지도 검색
- 공유 문구 복사 또는 공유 시트 열기
- 동일 후보로 새 밥판 생성

기대 효과:

- 사용자가 결과를 보고 앱을 닫는 대신 실제 이동으로 이어진다.
- `결정 카드 생성기`를 넘어 `점심 행동 도구`처럼 느껴진다.
- 지도/장소 기능을 무겁게 만들기 전에도 실사용 가치가 생긴다.

주의:

- 근처 맛집 추천이나 장소 저장은 5차 범위에서 제외한다.
- 현재 위치 기반 자동 검색은 권한 설명 부담이 커질 수 있으므로, 먼저 단순 검색 인텐트로 시작한다.

### 3. 권한 전 안내를 `수집하지 않는 것`과 `주변에 보내는 것`으로 분리

기존 개선안은 권한 표현을 친근하게 바꾸는 데 집중했다. 5차에서는 출시 준비 관점에서 더 명확한 신뢰 구조를 만든다.

사용자는 `위치 권한을 왜 달라고 하지?`뿐 아니라 `내 밥닉과 선택이 어디까지 보이지?`도 궁금해할 수 있다. 권한 요청 전 안내 또는 설정 도움말에 두 영역을 분리한다.

권장 구성:

- `밥크로스가 수집하지 않는 것`
  - 위치 기록을 저장하지 않음
  - 서버로 밥닉, 후보, 선택 결과를 보내지 않음
  - 광고/분석 SDK로 사용자 행동을 보내지 않음
- `밥판 참여자에게 전달되는 것`
  - 밥닉과 아바타
  - 메뉴 후보
  - 선택 결과 또는 득표 요약
  - 결과 확인용 밥해시/영수증 정보

기대 효과:

- Nearby 권한에 대한 불안이 줄어든다.
- Play Store Data safety 설명과 앱 내부 문구가 일관된다.
- 초대받은 사람이 선택 공개 여부를 더 신뢰한다.

구현 메모:

- 설정의 개인정보처리방침 진입 전 요약 카드로 넣거나, 첫 권한 요청 전 바텀시트로 제공한다.
- 긴 법적 문구보다 체크리스트 형태가 좋다.
- `서버 없음`은 장점이지만, 주변 기기 간 공유는 숨기지 않고 분명히 말한다.

### 4. 지난 결정 화면을 `다시 쓰는 재료함`으로 재정렬

지난 결정은 이미 존재하지만, 반복 사용을 만들려면 기록이 단순 보관소가 아니라 다음 밥판의 재료가 되어야 한다. 4차에서 제안된 `이 후보로 다시 열기`를 더 강하게 제품 구조로 올린다.

제안:

- 지난 결정 화면 상단에 `바로 다시 열기` 영역 추가
- 카드별 주요 액션을 `보기`보다 `다시 열기` 중심으로 배치
- 이번 주 2등 메뉴, 최근 우승 메뉴, 자주 등장한 후보를 빠르게 넣을 수 있게 한다.
- `또 이걸로 정할까요?`처럼 반복 사용을 가볍게 유도한다.

기대 효과:

- 사용자가 매번 후보를 새로 입력하지 않아도 된다.
- 자주 쓰는 팀일수록 앱이 점점 편해지는 느낌을 준다.
- 기록 화면의 방문 이유가 생긴다.

구현 메모:

- 새 통계 시스템보다 기존 기록 데이터에서 계산 가능한 범위로 시작한다.
- 상단 추천은 최대 3개만 보여준다.
- 메뉴명이 긴 경우 `후보 N개로 다시 열기`처럼 요약한다.

### 5. 공유 초대 문구를 상황별로 자동 선택

공유 이미지 안전 영역과 미리보기는 이미 좋아졌다. 다음은 대화방에 같이 붙는 문구의 맥락이다. 같은 결과라도 `혼자 먼저 정리`인지 `같이 고르기`인지에 따라 메시지가 달라야 한다.

상황별 문구:

- 혼자 먼저 정리: `후보 먼저 정리해봤어요. 이대로 갈까요?`
- 같이 고르기: `밥판 열었어요. 앱 켜고 1분 안에 골라줘요.`
- 동률 결과: `공동 1등이라 결승 밥판 열어도 좋겠어요.`
- 후보 추가 허용: `먹고 싶은 메뉴 있으면 후보에 하나 더 얹어줘요.`
- 비공개 선택: `누가 뭘 골랐는지는 결과에서 정한 방식대로만 보여요.`

기대 효과:

- 공유받은 사람이 앱을 왜 켜야 하는지 더 빨리 이해한다.
- 밥판장이 별도로 설명해야 하는 말이 줄어든다.
- 카카오톡/문자에서 밥크로스의 말투가 더 선명해진다.

구현 메모:

- 공유 이미지 생성 로직과 문구 생성 로직을 분리한다.
- 공유 문구는 짧게 유지하고, 후보 목록은 너무 길면 `후보 6개`처럼 요약한다.
- 사용자가 미리보기에서 문구를 복사하기 전에 확인할 수 있게 한다.

### 6. 출시용 스크린샷 시나리오와 단말 테스트 매트릭스 정리

기능 완성도가 올라갈수록 출시 준비는 `잘 된다`가 아니라 `어떤 상황에서 검증했다`로 관리해야 한다.

스크린샷 5장 제안:

1. 홈: `메뉴 못 정할 때 밥판 열기`
2. 밥판 열기: `혼자 먼저 정리하거나 같이 고르기`
3. 초대 화면: `이 밥판 규칙을 보고 바로 참여`
4. 결과 카드: `오늘의 밥결정 공유`
5. 설정/신뢰: `서버 없이 가까운 밥친구끼리`

단말 테스트 매트릭스:

- Android 12, 13, 14, 15 이상
- Samsung 2대 이상 동시 연결
- 권한 전체 허용
- 위치 권한 거부
- Bluetooth 꺼짐
- Wi-Fi 꺼짐
- 앱 설치 직후 첫 실행
- 앱 데이터 삭제 후 재실행
- 2명, 3명, 5명 밥판
- 긴 질문, 긴 메뉴명, 후보 10개 이상
- 동률 결과와 시간 만료 결과

기대 효과:

- Play Store 심사와 베타 테스트 대응이 쉬워진다.
- 릴리스 직전 회귀 테스트 범위가 명확해진다.
- 스토어 스크린샷이 기능 나열이 아니라 실제 사용 장면이 된다.

## 구현 순서

### 1단계: 초대 예측 가능성 개선

목표는 밥판장이 밥신호를 보내기 전에 상대 화면을 상상할 수 있게 하는 것이다.

작업:

1. 밥판 열기 화면에 `상대에게 이렇게 보여요` 카드 추가
2. 현재 질문, 후보 수, 제한시간, 공개 여부, 후보 추가 여부를 프리뷰에 반영
3. `혼자 먼저 정리` 모드에서는 결과 공유 프리뷰로 문구 전환
4. 긴 질문과 후보 과다 상태에서 레이아웃 확인

검증:

- 기본 템플릿, 직접 입력, 룰렛 후보 추가 상태에서 프리뷰 갱신 확인
- 긴 질문 2줄 처리 확인
- 화면 하단 주요 액션이 밀려나지 않는지 확인

### 2단계: 결과 후속 행동 강화

목표는 메뉴 결정 후 실제 점심 행동으로 이어지게 하는 것이다.

작업:

1. 결과 화면에 `이제 어디로 갈까` 액션 묶음 추가
2. 결정 메뉴 기반 지도 검색 인텐트 연결
3. 결과 공유 문구와 이미지 공유 액션 정리
4. 동일 후보로 새 밥판을 여는 재사용 액션 추가

검증:

- 일반 결과, 동률 결과, 혼자 결정 결과에서 액션 표시 확인
- 지도 앱이 없거나 인텐트 처리 앱이 없을 때 대체 토스트 확인
- 다시 열기 시 후보와 기본값이 의도대로 채워지는지 확인

### 3단계: 신뢰 안내 정리

목표는 권한과 데이터 공유 범위를 출시 가능한 언어로 설명하는 것이다.

작업:

1. 권한 요청 전 또는 설정에 `수집하지 않는 것` / `주변에 보내는 것` 요약 카드 추가
2. 개인정보처리방침 화면과 Play Store 등록 메모 문구 일치 여부 확인
3. 초대 규칙 카드의 선택 공개 문구와 신뢰 안내 표현 맞추기

검증:

- 첫 설치 후 권한 안내 흐름 확인
- 권한 거부 후 `밥판 체험하기` 또는 설정 이동이 자연스러운지 확인
- Play Store Data safety 설명과 충돌하는 표현이 없는지 확인

### 4단계: 반복 사용 동선 정리

목표는 지난 결정이 다음 밥판의 재료가 되게 하는 것이다.

작업:

1. 지난 결정 화면 상단에 `바로 다시 열기` 추천 영역 추가
2. 최근 우승 메뉴, 지난 2등 메뉴, 자주 나온 후보를 최대 3개 표시
3. 결과 카드별 `이 후보로 다시 열기` 액션을 더 선명하게 배치

검증:

- 기록이 없을 때 빈 상태 문구 확인
- 기록이 1개, 여러 개, 동률 포함일 때 추천 계산 확인
- 후보가 긴 경우 버튼과 카드가 깨지지 않는지 확인

### 5단계: 출시 준비 패키지 정리

목표는 베타 배포와 Play Store 등록 전에 필요한 증거를 남기는 것이다.

작업:

1. 스크린샷 5장 시나리오 확정
2. 단말 테스트 매트릭스 문서화
3. 권한 거부, 연결 실패, 시간 만료, 동률 결과 회귀 테스트
4. 릴리스 빌드 기준 `debuggable`, signing, versionName/versionCode 확인

검증:

- debug APK뿐 아니라 release AAB 기준 설치 또는 내부 테스트 확인
- 개인정보처리방침 URL 공개 접근 확인
- 스토어 설명과 앱 내부 문구의 표현 일치 확인

## 출시 전 체크리스트

- 밥판 열기 화면에서 상대방 프리뷰가 현재 입력값과 일치한다.
- 결과 화면에서 지도 검색, 공유, 다시 열기가 한 화면에서 자연스럽게 보인다.
- 권한 안내가 `수집하지 않는 것`과 `주변에 보내는 것`을 구분한다.
- 초대받은 사람은 선택 공개 여부와 후보 추가 가능 여부를 참여 전에 알 수 있다.
- 지난 결정 화면에서 다음 밥판을 바로 만들 수 있다.
- 공유 문구가 혼자 모드와 같이 고르기 모드에 맞게 달라진다.
- 긴 질문, 긴 메뉴명, 후보 10개 이상에서도 텍스트가 겹치지 않는다.
- 권한 거부, Bluetooth 꺼짐, Wi-Fi 꺼짐 상태에서 다음 행동이 보인다.
- Play Store 스크린샷 5장의 장면과 문구가 확정되어 있다.
- 릴리스 빌드와 Data safety 설명이 현재 앱 동작과 모순되지 않는다.

## 결론

5차 개선의 핵심은 기능을 더 얹는 것이 아니라, 이미 만든 밥판 흐름이 실제 점심 대화 안에서 덜 어색하게 작동하도록 마지막 연결부를 다듬는 것이다.

가장 먼저 `상대에게 이렇게 보여요` 프리뷰를 넣어 첫 사용자의 예측 가능성을 높이고, 이어서 결과 화면의 `이제 어디로 갈까` 행동 묶음을 붙이는 것이 좋다. 그 다음 권한/신뢰 안내와 지난 결정 재사용 동선을 정리하면, 밥크로스는 `귀여운 투표 앱`을 넘어 `한 명만 켜도 점심 결정을 짧게 만드는 도구`에 더 가까워진다.
