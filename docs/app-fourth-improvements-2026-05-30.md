# 밥크로스 4차 개선안

작성일: 2026-05-30

## 현재 기준

현재 밥크로스는 `0.1.26 / versionCode 27` 기준으로 다음 축이 어느 정도 갖춰졌다.

- 홈에서 `밥판 열기`, `밥판 체험하기`, 최근 결정 확인이 가능하다.
- 밥판 열기 화면에서 템플릿, 메뉴 룰렛, 지난 2등 메뉴 추천을 쓸 수 있다.
- 밥판장이 혼자 앱을 켜도 후보를 만들고 한 표를 고른 뒤 `오늘의 밥결정 만들기`까지 갈 수 있다.
- 결과 카드 공유 이미지에 메뉴, 후보, 결정 이유가 들어간다.
- 설정 화면에는 프로필 카드, 연결 준비 상태 카드, 자동 연결 토글형 카드가 들어갔다.

따라서 4차 개선의 방향은 새로운 기능을 크게 늘리기보다, 실제 점심 자리에서 `처음 켠 사람`, `초대받은 사람`, `공유받은 사람`이 각각 덜 망설이게 만드는 데 둔다.

## 핵심 방향

밥크로스의 다음 단계는 다음 문장으로 정리할 수 있다.

```text
한 명이 정리하고, 모두가 이해하고, 결과가 밖으로 퍼지는 밥결정 도구
```

1-3차 개선이 앱 안의 흐름을 정리했다면, 4차 개선은 앱 밖 대화방까지 포함한 사용 장면을 다듬어야 한다.

## 가장 중요한 사용자 장면

### 1. 밥판장이 혼자 먼저 여는 장면

밥판장은 대화가 길어지기 전에 앱을 켜고 후보를 정리한다. 이때 앱은 `다른 사람이 아직 안 들어왔다`를 문제처럼 보여주면 안 된다.

좋은 흐름:

1. 후보를 빠르게 만든다.
2. 내 선택을 넣는다.
3. 오늘의 밥결정 카드를 만든다.
4. 카카오톡, 문자, 메신저에 던진다.
5. 친구가 앱을 켜면 더 공정하게 같이 고를 수 있다는 메시지가 남는다.

### 2. 밥친구가 초대를 받는 장면

초대받은 사람은 앱을 켰을 때 권한, 연결, Nearby 같은 구조를 이해하고 싶지 않다. 지금 필요한 것은 `무슨 밥판인지`, `무엇을 누르면 되는지`, `내 선택이 공개되는지`다.

좋은 흐름:

1. 누가 보냈는지 본다.
2. 메뉴 후보를 훑는다.
3. 내 선택 공개 여부를 안다.
4. 바로 `밥판 참여` 또는 `거절`을 누른다.

### 3. 결과를 공유받는 장면

앱을 설치하지 않은 사람은 공유 이미지만 보고도 맥락을 이해해야 한다. 결과 카드가 바이럴 소재가 되려면 예쁘기만 해서는 부족하고, `왜 이 메뉴인지`가 한눈에 들어와야 한다.

좋은 공유 결과:

- 오늘의 메뉴가 가장 크게 보인다.
- 후보와 득표 요약이 보인다.
- 동률이면 추천 기준이 보인다.
- 앱 이름과 밥크로스 캐릭터가 기억에 남는다.

## 우선순위 높은 개선안

### 1. 밥판 열기 화면에 `오늘 바로 정하기` 모드 추가

현재 밥판 열기는 기본적으로 밥신호를 보내는 느낌이 강하다. 하지만 제품 방향은 `혼자 써도 가치가 보이는 앱`이다. 따라서 밥판 열기 화면에서 사용자가 지금 목적을 고를 수 있게 한다.

제안:

- 상단에 작은 세그먼트 컨트롤을 둔다.
- 옵션은 `혼자 먼저 정리`, `같이 고르기` 정도로 둔다.
- `혼자 먼저 정리`에서는 문구를 `후보를 정리하고 오늘의 밥결정을 만듭니다.`로 바꾼다.
- `같이 고르기`에서는 기존처럼 밥신호, 밥친구 참여, 제한시간을 강조한다.

기대 효과:

- 연결된 사람이 없어도 이상한 상태처럼 느껴지지 않는다.
- 밥판장이 앱을 켜는 첫 이유가 더 명확해진다.
- 이후 공유 카드 흐름과 자연스럽게 이어진다.

### 2. 결과 공유 전에 미리보기 화면 추가

현재는 결과 화면에서 공유 버튼을 누르면 바로 공유 시트로 넘어간다. 공유 이미지를 강화한 만큼, 사용자가 공유 전에 `이 이미지가 어떻게 보이는지` 확인할 수 있으면 품질 신뢰가 올라간다.

제안:

- `오늘의 밥결정 카드 공유`를 누르면 먼저 공유 미리보기 바텀시트를 연다.
- 미리보기에는 실제 공유 이미지 비율을 축소해 보여준다.
- 액션은 `이미지 공유`, `문구만 복사`, `다시 고르기`로 둔다.
- 공유 문구는 현재처럼 메뉴, 후보, 이유를 포함한다.

기대 효과:

- 이미지 잘림, 문구 과다, 동률 표현 문제를 사용자도 공유 전에 확인할 수 있다.
- 앱 밖으로 나가는 결과물의 완성도가 높아진다.
- 공유 실패 시에도 문구 복사로 대체할 수 있다.

### 3. 초대 카드에 `상대방에게 보이는 정보`를 명확히 표시

초대받은 사람에게 가장 중요한 정보는 선택 공개 여부와 후보 추가 가능 여부다. 현재도 일부 문구가 있지만, 카드형 요약으로 더 빠르게 이해하게 만들 수 있다.

제안:

- 밥판 초대 화면에 `이 밥판 규칙` 카드 추가
- 항목:
  - `내 선택 공개`: 공개 또는 득표수만 공개
  - `후보 추가`: 가능 또는 불가
  - `남은 시간`: 예: 4분 12초
- 참여 버튼 근처에 `한 번 고르면 바꿀 수 없어요`를 짧게 표시한다.

기대 효과:

- 참여 전 불안감이 줄어든다.
- 비공개 투표의 장점이 더 잘 보인다.
- 실제 모임에서 설명해야 하는 말이 줄어든다.

### 4. 설정에 밥판 기본값 섹션 추가

설정 1차 개선으로 프로필과 연결 상태는 좋아졌다. 다음은 매번 반복 조작하는 밥판 기본값을 줄이는 단계다.

제안:

- `밥판 기본값` 섹션 추가
- 기본 제한시간: 30초, 1분, 3분, 5분, 직접 설정
- 밥친구 후보 추가 기본값
- 결과에서 밥친구별 선택 공개 기본값
- 최근 사용 템플릿 바로가기

기대 효과:

- 자주 쓰는 사용자가 매번 같은 설정을 반복하지 않는다.
- 회식형, 점심형, 후식형 사용 습관을 앱이 기억하는 느낌이 생긴다.
- 설정 화면이 단순 연결 관리가 아니라 사용 방식 관리 화면이 된다.

### 5. 연결 문제 해결 화면을 일반 사용자 언어로 재구성

현재 내부에는 `개발자 진단`, `주변 연결 시작` 같은 기술 중심 표현이 남아 있다. 실제 사용자에게는 `밥친구를 못 찾을 때`로 보여야 한다.

제안:

- 설정 메인 진입명: `밥친구를 못 찾을 때`
- 화면 제목: `밥친구 찾기 점검`
- 주요 버튼:
  - `밥친구 다시 찾기`
  - `권한 다시 확인`
  - `설정 열기`
  - `밥판 체험하기`
- 로그는 접힌 `상세 기록` 안에 둔다.

기대 효과:

- 실패 상태가 개발자 도구처럼 보이지 않는다.
- 권한 문제가 있어도 사용자는 다음 행동을 바로 알 수 있다.
- 앱의 브랜드 언어가 설정/진단 영역까지 일관된다.

### 6. 지난 결정 화면을 `다음 밥판 만들기` 중심으로 재정렬

결과 기록은 단순 아카이브가 아니라 다음 결정을 빠르게 만드는 재료다. `지난 메뉴 결정` 화면은 재사용성이 더 강해야 한다.

제안:

- 각 결과 카드에 `이 후보로 다시 열기`를 더 선명하게 표시한다.
- 지난 2등 메뉴, 최근 1등 메뉴, 자주 나온 메뉴를 상단 추천으로 올린다.
- `이번 주 밥상 결산`에서 바로 `아쉬운 2등으로 밥판 열기`를 제공한다.

기대 효과:

- 기록 화면이 보는 화면에서 행동 화면으로 바뀐다.
- 매번 후보를 새로 만드는 부담이 줄어든다.
- 앱을 반복 실행할 이유가 생긴다.

## 추가 개선 아이디어

### 후보별 `싫어요` 또는 `빼기` 신호

말로는 싫다고 하기 애매한 메뉴를 앱 안에서 가볍게 제외할 수 있게 한다. 초기에는 복잡한 점수제보다 `이건 오늘 빼기` 정도가 좋다.

주의:

- 부정 감정이 너무 강하게 보이지 않게 표현한다.
- 결과 화면에서는 `제외 요청이 많았던 메뉴` 정도로 부드럽게 보여준다.

### 동률 결승전

공동 1등일 때 현재는 추천 멘트를 보여준다. 다음 단계에서는 `결승전 열기`를 제공할 수 있다.

제안:

- 동률 메뉴만 후보로 새 밥판 생성
- 제한시간은 30초 또는 1분으로 짧게 고정
- 결과 카드에는 `결승 밥판` 라벨 표시

### 지도/장소 흐름 강화

결정 메뉴가 나온 뒤 사용자는 결국 어디로 갈지 정해야 한다. 현재 지도 검색 연결은 좋은 시작점이다.

제안:

- `지도에서 찾기` 옆에 `근처 식당 정하기` 문구 사용
- 메뉴명만 검색하지 말고 `현재 위치 + 메뉴 맛집` 검색 의도를 명확히 안내
- 추후에는 즐겨찾는 식당 후보를 저장하는 방향 검토

### 공유 카드 테마

밥크로스의 결과 카드는 이미 차별성이 있다. 장기적으로는 테마가 반복 공유의 재미가 될 수 있다.

제안:

- 기본, 회식, 후식, 전설 카드 테마
- 희귀도 연출은 유지하되 정보 가독성을 우선한다.
- 공유 이미지는 카드 안쪽 안전 영역을 정의해 텍스트가 항상 들어오게 한다.

## 문구 정리 제안

피해야 할 표현:

- 주변 연결 시작
- 개발자 진단
- Nearby 권한
- 접속자 없이 게시
- 연결 준비 필요

권장 표현:

- 밥친구 다시 찾기
- 밥친구를 못 찾을 때
- 밥신호 보내기
- 혼자 먼저 밥판 열기
- 오늘의 밥결정 만들기
- 결과 카드 공유
- 권한 없이 먼저 체험하기

## 권장 1차 실행 순서

### 1단계: 공유 미리보기와 안전 영역 정리

가장 먼저 공유 결과물의 품질을 안정화한다. 공유 이미지는 앱 밖에서 밥크로스를 처음 보는 사람이 만나는 첫 화면이므로, 텍스트 잘림과 정보 과다를 막는 것이 중요하다.

작업:

1. 공유 미리보기 바텀시트 추가
2. 이미지 안전 영역 상수화
3. 후보/이유 텍스트 길이에 따른 자동 축약 또는 글자 크기 조정
4. `문구만 복사` 액션 추가

### 2단계: 밥판 열기 모드 분리

혼자 쓰는 가치와 같이 고르는 가치를 한 화면에서 자연스럽게 분리한다.

작업:

1. `혼자 먼저 정리` / `같이 고르기` 모드 추가
2. 모드별 안내 문구 변경
3. 접속자 없음 팝업과 상태 카드 문구 재정렬
4. 혼자 모드에서는 결과 공유 흐름을 더 강조

### 3단계: 초대와 규칙 카드 개선

초대받은 사람이 망설이지 않게 한다.

작업:

1. 밥판 초대 화면에 규칙 카드 추가
2. 선택 공개 여부와 후보 추가 가능 여부를 배지로 표시
3. 참여 후 변경 불가 안내 추가
4. 거절 후 홈으로 돌아가는 흐름 문구 점검

### 4단계: 설정 기본값과 진단 명칭 정리

반복 사용자의 조작 비용을 줄이고, 기술 표현을 사용자 언어로 치환한다.

작업:

1. 밥판 기본값 저장 구조 추가
2. 기본 제한시간, 후보 추가 허용, 선택 공개 기본값 적용
3. `개발자 진단`을 `밥친구 찾기 점검`으로 변경
4. 로그는 상세 영역으로 숨김

## 출시 전 품질 체크리스트

- 공유 이미지에서 모든 텍스트가 카드 안쪽 안전 영역에 들어오는지 확인
- 긴 질문, 긴 메뉴명, 후보 10개 이상, 공동 1등 상황에서 결과 카드 확인
- 권한 거부, Bluetooth 꺼짐, Wi-Fi 꺼짐, 위치 꺼짐 상태에서 홈/설정/초대 문구 확인
- 연결된 기기 0대, 1대, 3대 이상에서 밥판 생성과 결과 공유 확인
- 밥판장이 혼자 결정한 결과와 여러 명이 참여한 결과가 모두 기록에 남는지 확인
- 공유 이미지와 공유 문구가 카카오톡/문자/기본 공유 시트에서 읽기 좋은지 확인

## 결론

4차 개선의 초점은 밥크로스를 `앱 안에서만 잘 도는 투표 도구`가 아니라 `대화방까지 이어지는 밥결정 도구`로 만드는 것이다.

가장 먼저 공유 미리보기와 이미지 안전 영역을 안정화하고, 이어서 밥판 열기 화면을 `혼자 먼저 정리`와 `같이 고르기`로 나누는 것이 좋다. 그 다음 초대 규칙 카드와 설정 기본값을 붙이면, 첫 사용자와 반복 사용자 모두의 마찰이 줄어든다.
